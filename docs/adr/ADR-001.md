# ADR-001: Вибір СУБД для системи обліку СТО

## Статус
Accepted

## Контекст
Система призначена для локального використання на одному робочому комп'ютері адміністратора СТО або невеликому сервері.
Одночасна робота користувачів: 1-3 особи (Адмін + 2 Майстри).
Бюджет на інфраструктуру: 0 грн (потрібне безкоштовне рішення).
Дані структуровані, є чіткі зв'язки (1:N) між Замовленням та Послугами.

## Рішення
Використовувати **SQLite 3** як основну СУБД.

## Обґрунтування
1. **Нульова конфігурація:** SQLite не потребує встановлення окремого сервера БД (як PostgreSQL/MySQL), що спрощує розгортання на звичайному ПК СТО.
2. **Вбудована підтримка:** Django має повну вбудовану підтримку SQLite "з коробки".
3. **Продуктивність:** Для навантаження <5 користувачів SQLite працює швидше за клієнт-серверні СУБД завдяки відсутності мережевих затримок.
4. **Переносимість:** Вся база — це один файл `.sqlite3`, який легко копіювати для бекапу.

## Альтернативи
| Варіант | Чому відхилено |
|---------|---------------|
| PostgreSQL | Занадто складно налаштовувати та обслуговувати для такої малої системи. |
| MySQL | Потребує запуску демону бази даних, що зайве навантажує сервер СТО. |
