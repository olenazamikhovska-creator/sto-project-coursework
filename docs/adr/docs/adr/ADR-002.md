# ADR-002: Вибір технологічного стеку: Python + Django

## Статус
Accepted

## Контекст
Необхідно швидко розробити систему з можливістю адміністрування замовлень.
Команда знайома з мовою Python.
Потрібна швидка генерація інтерфейсу для введення даних (CRUD).

## Рішення
Використовувати фреймворк **Django (Python)** та ORM для роботи з даними.

## Обґрунтування
1. **Django Admin:** Вбудована адмін-панель дозволяє створити інтерфейс для Майстрів та Адміністратора за години, без написання HTML/JS коду вручну.
2. **Безпека:** Django автоматично захищає від SQL-ін'єкцій та XSS (критично для обліку клієнтів).
3. **Масштабованість:** Якщо СТО виросте, Django легко мігрувати на PostgreSQL без зміни коду бізнес-логіки.

## Альтернативи
| Варіант | Чому відхилено |
|---------|---------------|
| PHP (Laravel) | Команда не має достатнього досвіду з PHP; синтаксис Python зручніший для читання. |
| Node.js | Менш підходить для швидкої розробки класичних CRM-систем порівняно з Django. |
